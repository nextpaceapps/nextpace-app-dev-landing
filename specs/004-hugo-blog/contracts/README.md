# Contracts: Hugo-Based Blog Integrated at /blog Route

**Feature**: Hugo-Based Blog Integrated at /blog Route  
**Date**: 2025-01-27  
**Phase**: 1 - Design & Contracts

## Overview

This feature is a static site generated by Hugo with no backend API requirements. All blog content is stored as Markdown files and generated into static HTML at build time.

## API Contracts

**N/A** - No API endpoints required.

The Hugo blog:
- Does not fetch data from any API
- Does not require backend services
- Uses static content (Markdown files generated to HTML)
- Does not perform any data mutations
- Does not require authentication or authorization

## File Structure Contracts

### Hugo Content Structure

**Content Directory**: `blog/content/posts/`

**File Format**: Markdown (`.md`) files with front matter

**Front Matter Contract**:
```yaml
---
title: string          # Required: Post title
date: datetime         # Required: Publication date (YYYY-MM-DD or RFC3339)
slug: string          # Optional: URL slug (defaults to filename)
excerpt: string       # Optional: Preview text
author: string        # Optional: Author name
tags: []string        # Optional: Array of tags
categories: []string  # Optional: Array of categories
draft: boolean        # Optional: Draft status (default: false)
featured: boolean     # Optional: Featured post (default: false)
---
```

**Content Body**: Markdown format supporting:
- Headings (# ## ###)
- Paragraphs
- Lists (ordered, unordered)
- Links `[text](url)`
- Code blocks (```language)
- Images `![alt](path)`
- Bold/italic formatting

### Hugo Configuration Contract

**Config File**: `blog/config.toml` or `blog/hugo.toml`

**Required Configuration**:
```toml
baseURL = "https://nextpace.dev"
title = "Next Pace Dev Blog"
publishDir = "../dist/blog"
paginate = 5
theme = "neonic-blog"
[permalinks]
  posts = "/blog/:slug"
```

### Build Output Contract

**Output Directory**: `dist/blog/`

**Generated Files**:
- `index.html` - Blog listing page (at `/blog/`)
- `page/2/index.html`, `page/3/index.html` - Paginated listing pages
- `{post-slug}/index.html` - Individual post pages (at `/blog/{post-slug}/`)

**URL Mapping**:
- `/blog` → `dist/blog/index.html`
- `/blog/page/2` → `dist/blog/page/2/index.html`
- `/blog/{post-slug}` → `dist/blog/{post-slug}/index.html`

## Integration Contracts

### Header/Footer Integration

**Header Partial**: `blog/themes/neonic-blog/layouts/partials/header.html`
- Must match React Navbar component design
- Must use neonic theme colors and styling
- Must include navigation links (Services, Process, Pricing, Blog, Contact)
- Must be valid HTML5

**Footer Partial**: `blog/themes/neonic-blog/layouts/partials/footer.html`
- Must match React Footer component design
- Must use neonic theme colors and styling
- Must include footer columns (About Us, Projects, Solutions, Blog, Terms)
- Must be valid HTML5

### Styling Contract

**Theme Stylesheet**: `blog/themes/neonic-blog/static/css/main.scss`

**Required Design Tokens** (must match neonic theme):
- Colors: `$primary: #06b6d4`, `$secondary: #e879f9`, `$bg-main: #000000`, `$bg-secondary: #171717`, `$text-primary: #ffffff`
- Typography: Must match main website font family and sizes
- Spacing: Must use consistent spacing scale
- Breakpoints: `$breakpoint-md: 768px`, `$breakpoint-lg: 1024px`

**Note**: Current implementation uses static file generation. No API contracts are needed at this time.
