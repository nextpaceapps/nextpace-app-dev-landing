# Data Model: Hugo-Based Blog Integrated at /blog Route

**Feature**: Hugo-Based Blog Integrated at /blog Route  
**Date**: 2025-01-27  
**Phase**: 1 - Design & Contracts

## Overview

The Hugo blog uses Hugo's content management structure with Markdown files and front matter metadata. No database or API is required - all content is stored as static Markdown files and generated into HTML by Hugo.

## Entities

### Blog Post (Markdown File)

Represents a single blog post stored as a Markdown file in Hugo's content directory.

**Location**: `blog/content/posts/*.md`

**Front Matter Fields** (YAML/TOML in Markdown file header):
- `title: string` - Blog post title (required)
- `date: datetime` - Publication date (required, format: YYYY-MM-DD or RFC3339)
- `slug: string` - URL slug (optional, defaults to filename, format: `post-slug`)
- `excerpt: string` - Short preview/excerpt text (optional, can be auto-generated)
- `draft: boolean` - Whether post is draft (default: false)
- `author: string` - Author name (optional)
- `tags: []string` - Array of tags (optional)
- `categories: []string` - Array of categories (optional)
- `featured: boolean` - Whether post is featured (optional, default: false)

**Content**: Markdown body content (headings, paragraphs, lists, links, code blocks, images)

**Example**:
```markdown
---
title: "My First Blog Post"
date: 2025-01-27T10:00:00Z
slug: "my-first-blog-post"
excerpt: "This is a brief excerpt of the blog post."
author: "Next Pace Dev"
tags: ["development", "hugo"]
categories: ["tutorials"]
draft: false
---

# My First Blog Post

This is the content of the blog post in Markdown format...
```

**URL Structure**: `/blog/{slug}` (e.g., `/blog/my-first-blog-post`)

---

### Blog Listing Page

Represents the main blog page displaying paginated list of posts.

**Location**: Generated by Hugo from `layouts/_default/list.html` template

**Data Source**: Hugo's `.Site.RegularPages` filtered to blog posts, paginated

**Fields** (available in template):
- `Posts: []Page` - Array of blog post pages (paginated, 5 per page)
- `Paginator: Paginator` - Pagination object with page numbers, next/previous links
- `CurrentPage: int` - Current page number
- `TotalPages: int` - Total number of pages
- `HasPrev: boolean` - Whether previous page exists
- `HasNext: boolean` - Whether next page exists

**Pagination**: 5 posts per page, traditional pagination (page numbers or Next/Previous buttons)

---

### Blog Post Page

Represents an individual blog post page displaying full content.

**Location**: Generated by Hugo from `layouts/_default/single.html` template

**Data Source**: Single Markdown file from `content/posts/`

**Fields** (available in template):
- `.Title` - Post title from front matter
- `.Date` - Publication date from front matter
- `.Content` - Rendered Markdown content (HTML)
- `.Params.author` - Author name (if provided)
- `.Params.tags` - Tags array (if provided)
- `.Params.categories` - Categories array (if provided)
- `.Params.excerpt` - Excerpt text (if provided)
- `.Permalink` - Full URL to post (e.g., `/blog/my-first-post`)

---

### Hugo Site Configuration

Represents Hugo site configuration and settings.

**Location**: `blog/config.toml` or `blog/hugo.toml`

**Key Configuration Fields**:
- `baseURL: string` - Site base URL (e.g., "https://nextpace.dev")
- `title: string` - Site title
- `publishDir: string` - Output directory (e.g., "../dist/blog")
- `paginate: int` - Posts per page (5)
- `theme: string` - Theme name (e.g., "neonic-blog")
- `permalinks.posts: string` - URL pattern for posts (e.g., "/blog/:slug")

## Validation Rules

1. **Blog Post Front Matter**:
   - `title` must be non-empty string
   - `date` must be valid date/datetime
   - `slug` must be URL-safe (lowercase, hyphens, no spaces)
   - `excerpt` should be concise (typically 50-200 characters)
   - `draft: false` posts are published, `draft: true` posts are excluded from build

2. **URL Structure**:
   - All post URLs must follow pattern `/blog/{slug}`
   - Slugs must be unique within the blog
   - Slugs should be descriptive and URL-friendly

3. **Pagination**:
   - Blog listing page displays maximum 5 posts per page
   - Pagination appears when total posts > 5
   - Page numbers or Next/Previous navigation must be functional

4. **Content Formatting**:
   - Markdown content must render correctly (headings, paragraphs, lists, links, code blocks, images)
   - Images must be properly referenced and accessible
   - Code blocks must have proper syntax highlighting if applicable

---

## State Transitions

**Blog Post Lifecycle**:
1. **Draft** (`draft: true`) → Not included in build, not accessible via URL
2. **Published** (`draft: false`) → Included in build, accessible via `/blog/{slug}` URL
3. **Future-dated** (`date` in future) → Can be included/excluded based on Hugo config (typically excluded)

**Build Process**:
1. Hugo reads Markdown files from `content/posts/`
2. Hugo processes front matter and content
3. Hugo generates static HTML files using templates
4. Hugo outputs to `dist/blog/` directory
5. Static HTML files are served by hosting platform

---

## Future Extensibility

1. **Taxonomies**: Hugo supports tags and categories natively - can be extended with taxonomy pages
2. **RSS Feed**: Hugo generates RSS feed automatically - can be enabled in config
3. **Search**: Can be added via client-side JavaScript or external search service
4. **Comments**: Can integrate third-party comment systems (Disqus, etc.)
5. **Related Posts**: Hugo can generate related posts based on tags/categories
6. **Author Pages**: Can create author archive pages if author field is used

---

## Notes

- All content is stored as Markdown files (no database)
- Hugo generates static HTML at build time (no runtime content generation)
- Front matter metadata controls post behavior and display
- Content structure follows Hugo conventions (`content/posts/` directory)
- URL structure is flat (`/blog/post-slug`) as specified
- Pagination is handled by Hugo's built-in pagination feature
